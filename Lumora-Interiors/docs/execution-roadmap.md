# Execution Roadmap

## Strategic Goal

Build Lumora Interiors as a premium digital showroom, not a generic service website. Recruiters should instantly see product thinking, visual discipline, conversion strategy, and implementation maturity.

## Competitive Advantage

Most internship submissions fail in five predictable ways: weak typography, random spacing, generic copy, inconsistent imagery, and no conversion architecture. Lumora wins by acting like a funded studio site:

- First screen sells the brand emotionally.
- Project pages prove thinking, not just decoration.
- Services are positioned as premium outcomes.
- Motion is subtle and expensive.
- Performance settings are documented and intentional.

## Phase 1: Foundation

- Theme: Hello Elementor for clean output and maximum control.
- Global colors: Matte Black `#111111`, Warm White `#F8F5F2`, Luxury Gold `#C8A96B`, Soft Gray `#D9D9D9`.
- Fonts: Playfair Display for headings, Inter for body and UI.
- Layout system: 1200px max content width, 12-column visual grid, 80px desktop section padding, 56px tablet, 40px mobile.
- Asset direction: editorial interiors, architectural details, warm natural light, premium material close-ups.

## Phase 2: Homepage Build

Build the homepage first because it defines the whole system. Use it as the visual benchmark for every other page.

1. Cinematic hero with dark overlay and two CTAs.
2. Trust stats with counters.
3. Featured projects grid.
4. Services cards.
5. Before/after transformation.
6. Design process timeline.
7. Testimonials carousel.
8. Masonry gallery.
9. FAQ accordion.
10. Consultation banner.
11. Footer.

## Phase 3: Content Pages

Create About, Services, Portfolio, Project Showcase, Testimonials, Blog, Contact, and Book Consultation. Use shared components: page hero, CTA band, footer, buttons, cards, and forms.

## Phase 4: Conversion System

Primary conversion: Book Consultation.

Secondary conversions:

- View Portfolio
- Explore Services
- Contact Studio
- Download design brief checklist from newsletter

Every page should have one dominant next action.

## Phase 5: Performance and QA

- Compress images to WebP, max 220KB for large images when possible.
- Use lazy loading below the fold.
- Preload hero image.
- Disable unused Elementor font icons where safe.
- Minify CSS and JS with LiteSpeed Cache.
- Test mobile spacing manually.

## Phase 6: Submission Polish

- Capture homepage desktop and mobile screenshots.
- Capture one portfolio detail screenshot.
- Record a 90-second demo video.
- Include README, setup guide, performance report, and deployment guide.
- Present the work as a premium WordPress build system, not just a page.
