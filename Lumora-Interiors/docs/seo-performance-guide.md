# SEO and Performance Guide

Book Consultation:

## Heading Rules

- One H1 per page.
- H2 for major sections.
- H3 for cards and sub-sections.
- Avoid using headings only for visual size.

## Image Rules

- Use WebP where possible.
- Hero image: 2200px wide, under 400KB if practical.
- Section images: 1400px wide, under 250KB.
- Card images: 900px wide, under 150KB.
- Use descriptive alt text: `Warm modern villa living room with stone wall and walnut furniture`.

## Performance Checklist

- Use Hello Elementor.
- Keep plugins minimal.
- Remove unused demo widgets and templates.
- Compress all images before upload.
- Enable lazy loading below the fold.
- Preload hero image.
- Use system fallback fonts while Google Fonts load.
- Avoid autoplay video on mobile unless compressed and muted.
- Use no more than two font families.
- Keep animation durations below 700ms.
- Test with Chrome Lighthouse and PageSpeed Insights.

## Accessibility Checklist

- All buttons and links have clear labels.
- Text contrast passes on dark overlays.
- Forms have labels, not placeholders only.
- Focus states remain visible.
- Accordions are keyboard accessible.
- Motion is subtle and does not block content.

## Lighthouse Target

- Performance: 90+
- Accessibility: 95+
- Best Practices: 95+
- SEO: 95+

## Common Fixes

- If LCP is poor, compress or replace hero media.
- If CLS appears, set image dimensions and aspect ratios.
- If JS execution is high, disable unused Elementor widgets and plugin modules.
- If accessibility drops, check contrast on gold text and overlay sections.
